The emacs_raw module¶

The emacs_raw module is implemented in C and provides a minimal interface for interfacing with Emacs.

Emacs objects¶

class emacs_raw.EmacsObject(obj, prefer_symbol=False, require_symbol=False)¶

Creates an Emacs lisp object from obj, according to the following rules.

  1. If its argument is an EmacsObject it will wrap the same underlying Emacs object.

    Note

    This creates a new Python object that wraps the same Emacs object, however EmacsObject instances are immutable so that hardly matters.

  2. If the argument has an __emacs__ magic method, it is called. This method must return an EmacsObject instance, and it must also accept the prefer_symbol keyword argument.

    Note

    It is up to the implementation of each class to decide on the appropriate meaning of prefer_symbol in this case.

  3. None and False become nil, while True becomes t.

  4. Integers and floating point numbers become their obvious Emacs lisp counterparts.

  5. Strings become strings unless prefer_symbol is true, in which case they become symbols.

  6. Lists and tuples become lists, the elements of which are coerced recursively.

  7. Anything else yields an error.

Note

Since Emacs lisp has so many different structures for mappings, there is no automatic coercion for dicts.

If require_symbol is true and the coerced object is not a symbol, an error is thrown.

Objects of type EmacsObject wrap any kind of Emacs lisp object. As a rule, primitives behave like their corresponding Python primitives. Arithmetic with numbers should work as expected, as well as adding strings. Every Emacs object except nil is truthy when in a boolean context.

Note

All such operations return the expected Python types, not new EmacsObject objects.

Note

Some behaviour is not yet supported, including string indexing, iterators (e.g. over the characters in a string, or elements of a list), or adding lists.

The repr of an EmacsObject is typically a string that can be passed to Emacs lisp read to yield an equivalent object. The str of an EmacsObject is the same, except without quotes in case the wrapped object actually is a string.

Emacs objects are also callable, and behave as expected if the wrapped object is a function. Arguments are automatically coerced to Emacs objects according to the above rules (with prefer_symbol false). Keyword arguments are also supported as follows:

obj(1, 'str', kw_arg=1.2)  # => (obj 1 "str" :kw-arg 1.2)
is_a(type)¶

Check whether the object has the given Emacs type.

Equivalent to (eq (type-of obj) (intern type)) in elisp.

type()¶

Return a string representation of the Emacs type of this object.

Equivalent to (symbol-name (type-of obj)) in elisp.

Custom constructors¶

emacs_raw.intern(name)

Creates an EmacsObject corresponding to the interned symbol with the given name. Equivalent to (intern name) in elisp.

emacs_raw.str(s)

Creates an EmacsObject of string type.

emacs_raw.int(i)

Creates an EmacsObject of integer type.

emacs_raw.float(f)

Creates an EmacsObject of floating point type.

emacs_raw.function(callback, nargs_min=0, nargs_max=PTRDIFF_MAX)

Creates an Emacs function object with the specificed arity which, when called, will run the given callback function and return its value to the Emacs caller, provided the return value is coercible to an EmacsObject.

If exceptions of type Signal or Throw are raised with appropriate arguments, the corresponding effect will be propagated to Emacs. All other exceptions are propagated as error signals.

emacs_raw.cons(car=None, cdr=None)

Creates an EmacsObject of cons type. If the car is not given, the return value is nil. The cdr defaults to nil.

emacs_raw.list(iterable)

Creates an EmacsObject of list (cons) type. All elements of the iterable must be Emacs objects.

emacs_raw.vector(iterable)

Creates an EmacsObject of vector type. All elements of the iterable must be Emacs objects.

Exceptions¶

exception emacs_raw.Signal(symbol, data)

Represents an Emacs signal (errors, mostly). When raised from Emacs, will have two arguments, a symbol denoting the type of signal (usually error, but not always), and a list with data, usually a single string giving an error message.

exception emacs_raw.Throw(tag, value)

Represents an Emacs throw (using the throw built-in). When raised from Emacs, will have two arguments, a symbol denoting the tag and an arbitrary object denoting the value.

Type checking¶

Warning

The following functions always return false for non-Emacs objects, even Python objects that behave as those types. Consider whether EAFP is for you.

emacs_raw.stringp(obj)

Check whether the object is an Emacs string. Equivalent to (stringp obj) in elisp.

emacs_raw.integerp()

Check whether the object is an Emacs integer. Equivalent to (integerp obj) in elisp.

emacs_raw.floatp(obj)

Check whether the object is an Emacs float. Equivalent to (floatp obj) in elisp.

emacs_raw.numberp(obj)

Check whether the object is an Emacs number (integer or float). Equivalent to (numberp obj) in elisp.

emacs_raw.number_or_marker_p(obj)

Check whether the object is an Emacs number or a marker. Equivalent to (number-or-marker-p obj) in elisp.

emacs_raw.symbolp(obj)

Check whether the object is an Emacs symbol. Equivalent to (symbolp obj) in elisp.

emacs_raw.consp(obj)

Check whether the object is an Emacs cons cell. Equivalent to (consp obj) in elisp.

emacs_raw.vectorp(obj)

Check whether the object is an Emacs vector. Equivalent to (vectorp obj) in elisp.

emacs_raw.functionp(obj)

Check whether the object is an Emacs function. Equivalent to (functionp obj) in elisp.

Comparison¶

Warning

The following functions fail for non-Emacs objects. Instead you should probably use the comparison operators directly, which is supported by the EmacsObject type to behave in a way that is consistent with Python types and their established behaviour.

emacs_raw.eq(a, b)

Check object equality using eq. Equivalent to (eq a b) in elisp.

emacs_raw.eql(a, b)

Check object equality using eql. Equivalent to (eql a b) in elisp.

emacs_raw.equal(a, b)

Check object equality using equal. Equivalent to (equal a b) in elisp.

emacs_raw.equal_sign(a, b)

Check object equality using =. Equivalent to (= a b) in elisp.

emacs_raw.string_equal(a, b)

Check object equality using string-equal. Equivalent to (string-equal a b) in elisp.

emacs_raw.lt(a, b)

Equivalent to (< a b) in elisp.

emacs_raw.le(a, b)

Equivalent to (<= a b) in elisp.

emacs_raw.gt(a, b)

Equivalent to (> a b) in elisp.

emacs_raw.ge(a, b)

Equivalent to (>= a b) in elisp.

emacs_raw.string_lt(a, b)

Equivalent to (string< a b) in elisp.

emacs_raw.string_gt(a, b)

Equivalent to (string> a b) in elisp.
