Launching TripoliÂ¶

To load the library from within Emacs, first make sure that the path where the library is installed is in your load-path. Then (require 'libtripoli) should suffice.

Once this is done, some functions should be available on the Emacs side.

  • (tripoli-import MODULE) imports a Python module by name.
  • (tripoli-exec-str CODE) runs Python code in the form of a string.
  • (tripoli-exec-buffer) runs the Python code in the current buffer.
  • (tripoli-exec-file FILE) runs a given Python file.
  • (tripoli-repl) runs a Python REPL in the terminal.
  • (tripoli-test &rest ARGS) runs the tests.

When loaded, Tripoli will run one of the files ~/.emacs.py or ~/.emacs.d/init.py if present. You can inhibit this behavior by binding tripoli-inhibit-init to a non-nil value before loading Tripoli.
